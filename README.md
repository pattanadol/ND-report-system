# ND Report System 📋

## นี่คืออะไร?

เว็บไซต์สำหรับแจ้งปัญหาต่างๆ ในองค์กร เหมือนกับการส่งใบแจ้งซ่อม แต่ทำผ่านเว็บให้สะดวกและติดตามได้

## ใช้เทคโนโลยีอะไร?

- **Next.js** - เฟรมเวิร์คสำหรับสร้างเว็บไซต์
- **TypeScript** - JavaScript แต่ปลอดภัยกว่า
- **React** - สร้าง UI ที่ใช้งานง่าย
- **Tailwind CSS** - จัดหน้าตาให้สวย
- **Firebase** - Database และ Authentication

## ระบบมีอะไรบ้าง?

### สำหรับผู้ดูแล (Admin)

- ดูเรื่องแจ้งทั้งหมด
- เปลี่ยนสถานะ (รอรับเรื่อง → กำลังทำ → เสร็จแล้ว)
- ลบเรื่องที่ไม่เหมาะสม
- ดูสถิติภาพรวม

### สำหรับผู้ใช้งาน (User)

- แจ้งปัญหาใหม่
- ดูเรื่องแจ้งของตัวเอง
- ติดตามว่าแก้ไขแล้วหรือยัง

## วิธีใช้งาน

1. เข้าเว็บไซต์
2. สมัครสมาชิก หรือ ล็อกอิน
3. แจ้งปัญหาผ่านฟอร์ม
4. รอการแก้ไขจากทีมงาน

## การติดตั้งและรัน

### 1. โคลนโปรเจ็กต์

```bash
git clone https://github.com/pattanadol/ND-report-system.git
cd ND-report-system
```

### 2. ติดตั้ง Dependencies

```bash
npm install
```

### 3. ตั้งค่า Firebase

1. ไปที่ [Firebase Console](https://console.firebase.google.com)
2. สร้างโปรเจ็กต์ใหม่
3. เปิดใช้งาน **Authentication** (Email/Password)
4. เปิดใช้งาน **Firestore Database**
5. คัดลอก Firebase config

### 4. ตั้งค่า Environment Variables

```bash
# คัดลอกไฟล์ตัวอย่าง
cp .env.example .env.local

# แก้ไขไฟล์ .env.local ใส่ Firebase config ของคุณ
```

### 5. รันโปรเจ็กต์

```bash
npm run dev
```

### 6. ตั้งค่าเริ่มต้น

1. เข้า `http://localhost:3000`
2. ไปที่หน้า Setup (`/setup`)
3. ทำตามขั้นตอนเพื่อสร้าง Admin User

### 7. เริ่มใช้งาน! 🎉

- ล็อกอินด้วย Admin account ที่สร้าง
- เริ่มใช้งานระบบแจ้งปัญหา

## โครงสร้างไฟล์หลักๆ

```text
src/
├── app/
│   ├── login/           # หน้าล็อกอิน
│   ├── register/        # หน้าสมัครสมาชิก
│   ├── setup/           # หน้าตั้งค่าเริ่มต้นระบบ
│   └── dashboard/       # หน้าหลักหลังล็อกอิน
│       ├── create/      # แจ้งเรื่องใหม่
│       ├── reports/     # จัดการเรื่องแจ้ง (Admin)
│       └── user/        # หน้าสำหรับ User
├── lib/
│   └── firebase/        # การเชื่อมต่อ Firebase
│       └── config.ts
├── utils/
│   ├── authContext.tsx  # จัดการการล็อกอิน
│   └── reportsContext.tsx # จัดการข้อมูลเรื่องแจ้ง
└── types/               # กำหนดโครงสร้างข้อมูล
```
